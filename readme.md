### 2. Загальні особливості та застереження

1. **Користувацькі нюанси**
   Інструкції описують робочу логіку скриптів на рівні практичного розуміння/використання, — а не на рівні технічної специфікації внутрішньої реалізації.
   Це — ЗАГАЛЬНА ІНСТРУКЦІЯ, до проекту. ІНСТРУКЦІЯ ДО ОБ'ЄКТУ, конкретного виконавчого файлу — лежить в тій же директорії та з назвою, що й сам об'єкт.
   При рівних ситуаціях, швидкість та якість — напряму залежить від характеристик потужності вашого ПК, компонентів та чистоти ваших даних.

2. **Робота з відкритими файлами**
   Перед запуском скриптів, що працюють із документами, рекомендовано закрити усі відповідні файли (Word, Excel, PDF, тощо) з якими буде працювати даний код.
   Це пов’язано з обмеженнями Windows: чатання/запису файлу, різними процесами водночас.
   Наслідки: некоректна обробка, відмова в доступі, помилки або пошкодження файлу.

4. **Старі формати**
   Старі формати файлів (наприклад, *.DOC, *.XLS) можуть читатися, але запис — у них не гарантується.
   Рекомендується попередньо конвертувати їх у сучасні формати (*.DOCX, *.XLSX).

5. **Уніфікований backup-підхід**
   Перед внесенням будь-який змін у файл — код перед цим ОБОВ'ЯЗКОВО (!) створює його копію у 'Кошику', з додаванням мітки дати та часу.
   Це дозволяє, за потреби відновити оригінал та 'переглянути' історію — без накопичення зайвих копій у робочих каталогах.

6. **Зручний виклик**
   Для централізованого запуску кількох виконавчих файлів рекомендуємо використовувати лаунчер проєкту: .launcher.py / .launcher.exe (за потреби файл можна перейменувати).
   Для одиничних запусків достатньо розмістити виконавчі файли у зручному для вас місці або в системних директоріях, таких як C:\Windows чи %USERPROFILE%, щоб за необхідності мати можливість швидкого виклику необхідних виконавчних файлів через Win + R.
