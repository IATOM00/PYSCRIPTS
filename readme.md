# PYSCRIPTS

Набір локальних Python-утиліт для Windows: запуск службових скриптів, робота з офісними документами, синхронізація даних, підготовка звітів та допоміжні системні операції.

Цей README описує загальні правила запуску та коротку карту сценаріїв. Детальна інструкція для конкретного скрипта, якщо вона потрібна, розміщується поруч із відповідним '.py' / '.exe' файлом.

## Вимоги

- Windows.
- Microsoft Office рекомендовано для скриптів, які працюють із документами Word / Excel.
- Для запуску готових '.exe' файлів Python-залежності не потрібні: необхідні ресурси вже включені до збірки.
- Для запуску '.py' файлів потрібен Python 3.11 або новіший та залежності, які використовує конкретний скрипт.

Базовий набір залежностей:

'''powershell
pip install pywin32 pandas numpy matplotlib openpyxl send2trash xlsxwriter python-docx pdfplumber pypdf
'''

## Запуск

Для централізованого запуску кількох утиліт використовуйте лаунчер проєкту:

- '.launcher.py'
- '.launcher.exe'

Для одиничного запуску можна використовувати відповідний '.py' або '.exe' файл напряму. За потреби виконавчі файли можна розмістити у зручному місці або в системних директоріях, наприклад 'C:\Windows' чи '%USERPROFILE%', щоб швидко запускати їх через 'Win + R'.

## Важливі застереження

### Робота з відкритими файлами

Перед запуском скриптів, які працюють із документами, рекомендовано закрити відповідні файли Word, Excel, PDF тощо. Якщо файл відкритий іншим процесом, Windows може заблокувати читання або запис, що призведе до помилки доступу, некоректної обробки або пошкодження файлу.

### Старі формати

Старі формати файлів, наприклад '.DOC' або '.XLS', можуть читатися, але коректний запис у них не гарантується. Перед обробкою бажано конвертувати такі файли у сучасні формати '.DOCX' або '.XLSX'.

### Backup-підхід

Скрипти, які вносять зміни у файли, перед записом створюють резервну копію оригіналу з міткою дати та часу й переміщують її до Кошика. Це дозволяє відновити попередню версію без накопичення зайвих копій у робочих каталогах.

### Продуктивність

Швидкість і якість обробки залежать від характеристик ПК, стану системи та якості вхідних даних.
